# 局域网媒体浏览器功能需求文档

## 一、项目背景

当前应用已具备局域网同步能力：

* 服务端通过 UDP 广播自身 IP 和 TCP 监听端口。
* 客户端自动发现并建立连接。
* 建立连接后保存服务端信息。
* 目前已实现设置项同步功能。
* 同步能力通过 HTTP 接口完成。

本需求在现有同步能力基础上新增：

**局域网媒体浏览器（Media Browser）**

目标用户仅为开发者本人。

暂不考虑：

* 多用户
* 权限系统
* 登录系统
* Linux 服务端
* macOS 服务端
* iOS 客户端
* HLS
* 视频转码
* 云端访问

仅支持：

* Windows 服务端
* Android 客户端

---

# 二、总体架构

客户端通过已有服务发现机制获得服务端地址。

所有媒体能力通过 HTTP API 实现。

架构：

客户端(Android)
↓
HTTP
↓
服务端(Windows)
↓
本地文件系统

视频播放直接使用 HTTP Range。

不使用 HLS。

---

# 三、服务端需求

## 3.1 根目录配置

服务端允许用户选择一个媒体根目录。

例如：

D:\Media

客户端看到的逻辑根目录统一表示为：

/

映射关系：

/
→ D:\Media

/sister
→ D:\Media\sister

/sister/video
→ D:\Media\sister\video

---

## 3.2 安全要求

必须防止路径穿越攻击。

禁止访问：

../../
....
等路径。

任何客户端请求最终解析后的实际路径必须满足：

resolvedPath.startsWith(rootDirectory)

否则返回错误。

绝对路径禁止暴露给客户端。

客户端仅能看到相对根目录路径。

---

## 3.3 文件项模型

文件夹与文件统一抽象为 FileItem。

字段：

```json
{
  "type": "directory",
  "name": "video",
  "relativePath": "/sister/video",
  "size": 0,
  "lastModified": 1712345678,
  "thumbnailUrl": null
}
```

```json
{
  "type": "file",
  "name": "cat.mp4",
  "relativePath": "/sister/video/cat.mp4",
  "size": 123456789,
  "lastModified": 1712345678,
  "mimeType": "video/mp4",
  "thumbnailUrl": "/api/media/thumbnail/sister/video/cat.mp4"
}
```

relativePath 为唯一标识。

客户端禁止依赖文件名作为唯一标识。

---

## 3.4 文件类型识别

图片：

jpg
jpeg
png
webp
gif

视频：

mp4
mkv
mov
avi
webm

其他类型统一视为普通文件。

---

## 3.5 文件列表接口

获取目录内容：

GET /api/media/list/{path}

示例：

GET /api/media/list/
GET /api/media/list/sister
GET /api/media/list/sister/video

返回当前目录下所有文件项。

排序规则：

第一优先级：
directory 在前
file 在后

第二优先级：
name 升序排序

示例：

folderA
folderB
folderC
aaa.mp4
bbb.mp4
ccc.mp4

---

## 3.6 缩略图接口

接口：

GET /api/media/thumbnail/{path}

禁止在文件列表接口中直接返回 Base64 图片。

文件列表仅返回 thumbnailUrl。

客户端自行懒加载。

---

## 3.7 缩略图生成策略

图片：

读取原图
缩放生成缩略图

视频：

视频时长 < 10 秒：

取中间帧

视频时长 ≥ 10 秒：

取第 5 秒帧

生成 JPG 缩略图。

禁止随机取帧。

---

## 3.8 缩略图缓存

服务端维护缩略图缓存目录：

.cache/thumbnails/

首次请求：

生成缩略图
写入缓存

后续请求：

直接返回缓存

避免重复调用 ffmpeg。

---

## 3.9 图片访问接口

GET /api/media/image/{path}

返回原图。

客户端负责缩放。

---

## 3.10 视频访问接口

GET /api/media/video/{path}

支持：

HTTP Range Request

必须正确返回：

206 Partial Content

支持 seek。

不进行转码。

直接返回文件内容。

# 新增约束

## 平台支持

本功能采用单代码库实现。

支持平台：

```text
服务端：Windows
客户端：Android
```

禁止为：

```text
Linux
macOS
iOS
Web
```

专门开发适配逻辑。

涉及平台差异时：

* 使用 Flutter 平台判断
* Windows 专属能力仅在 Windows 侧启用
* Android 专属能力仅在 Android 侧启用

避免无意义跨平台兼容代码。

---

## 中文路径支持

系统必须完整支持 Unicode 路径。

以下路径均应正常工作：

```text
/妹妹/视频/日常.mp4

/动画/葬送的芙莉莲/01.mp4

/写真/测试图片.jpg

/中文 空格 特殊符号/文件.mp4
```

要求：

* HTTP 接口正确 URL Encode / Decode
* 文件系统访问使用 UTF-8
* Flutter UI 正确显示中文文件名
* 缩略图接口支持中文路径
* 图片接口支持中文路径
* 视频接口支持中文路径

禁止因编码问题导致：

```text
404
乱码
无法播放
无法生成缩略图
```

---

## 缩略图缓存目录

缩略图缓存固定位置：

```text
{应用可执行文件目录}/.cache/thumbnails/
```

例如：

```text
MyApp.exe

.cache/
└── thumbnails/
    ├── a1b2c3.jpg
    ├── d4e5f6.jpg
    └── ...
```

禁止缓存到：

```text
用户目录
AppData
临时目录
媒体目录
```

缓存目录应由程序自动创建。

不存在时自动创建。

---

## 媒体浏览器入口

媒体浏览器不是独立页面。

必须作为：

```text
同步页
 ├── 连接
 ├── 同步
 └── 媒体浏览器
```

中的一个页签(Tab)实现。

与现有同步功能保持同级。

不得新增独立一级导航入口。

---

## 视频播放器中央提示区域

中央提示区域包括：

```text
暂停图标
快进提示
快退提示
Seek提示
倍速提示
```

设计要求：

* 居中显示
* 半透明背景
* 尽量减少对视频画面的遮挡
* 在没有手势、视频不暂停的情况下，也需要隐藏

推荐：

```text
圆角矩形

背景透明度：
20%~35%

尺寸仅包裹内容
```

禁止：

```text
全宽背景
高透明度遮罩
大面积黑色蒙层
```

目标效果类似：

```text
      ┌──────┐
      │ >>15 │
      └──────┘
```

而非：

```text
██████████████
████ >>15 ████
██████████████
```

保证观看视频时画面主体尽可能可见。

### 缩略图缓存 Key

不要用文件名。

例如：

```text
cat.mp4
```

生成：

```text
cat.jpg
```

会出问题。

建议明确：

```text
MD5(relativePath + fileSize + lastModified)
```

作为缓存文件名。

这样：

* 重名文件不会冲突
* 文件更新后自动失效
* 不需要额外数据库
